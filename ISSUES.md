# Issues et gestion des tâches

Ce dépôt V2 ne contient pas l'historique des issues GitHub du projet original.

## Comment retrouver ou migrer les issues ?
- **Option 1 :** Recréez manuellement les issues importantes dans l'onglet "Issues" de votre nouveau dépôt GitHub.
- **Option 2 :** Utilisez un script ou un outil d'export/import d'issues (GitHub API, ZenHub, etc.) pour migrer automatiquement les tickets.
- **Option 3 :** Copiez/collez les descriptions clés depuis l'ancien dépôt.

## Bonnes pratiques
- Utilisez des labels, des milestones et assignez les membres de l'équipe.
- Liez les issues aux Pull Requests pour un suivi optimal.
